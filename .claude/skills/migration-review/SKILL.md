# Migration Review

## When to use
하나의 구현체에서 다른 구현체로의 마이그레이션 작업이 일차적으로 완료된 시점에, 코드 변경 내역을 리뷰하고 피드백을 제공하기 위해 사용합니다.

## Workflow
1. **규칙 점검**: 루트 `AGENTS.md` 및 해당 디렉터리의 `AGENTS.md` 정책 준수 여부를 확인합니다.
2. **비교 검증**: 원본 레거시 구현체 대비 기능 동등성(Feature Parity), 아키텍처적 차이를 리뷰합니다.
3. **위험 요소 파악**: 
   - 파괴적인 변경(Destructive Change) 유무 확인
   - 중요한 에러 처리, 인증/인가 로직의 누락 여부 확인
4. **테스트/검증 점검**: 작성된 코드에 대한 단위/통합 테스트나 수동 검증 가이드가 적절히 포함되었는지 평가합니다.
5. **분류 및 리포팅**: 리뷰 결과를 중요도에 따라 분류합니다.

## Rules
- 기능이 원본과 다르게 동작할 가능성이 있다면 반드시 지적해야 합니다.
- 단순 코드 스타일의 문제보다 "아키텍처 목적"과 "기능 동등성"에 집중해야 합니다.

## Output
- 리뷰 결과를 다음 중요도로 분류하여 보고:
  - **Critical**: 치명적 오류, 기능 누락, 심각한 보안 이슈
  - **Important**: 아키텍처 불일치, 주요 예외 처리 누락
  - **Minor**: 코드 스타일, 네이밍 개선 권장사항
  - **Question**: 작업 의도를 명확히 파악하기 위한 질문 사항
